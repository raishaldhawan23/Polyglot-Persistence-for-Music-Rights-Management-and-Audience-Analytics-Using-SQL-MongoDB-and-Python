# Sonic Hub Studios: Building a Polyglot Database Architecture for Music Rights and Audience Analytics

## 1. Project Background
### Business Context:
The music industry generates two fundamentally different types of data.

On one side, companies need to manage structured business information such as music catalogues, client information, reservations, royalty payments, collaboration agreements and copyright claims. This information requires strong consistency and transactional reliability.

On the other hand, music companies increasingly rely on audience behaviour data generated from social media platforms, including engagement metrics such as likes, shares, comments, views and audience feedback. This data is dynamic, semi-structured and grows rapidly over time.

In this project, Sonic Hub Studios, a leading music company, collaborates with a social media influencer to feature personalised music in their content, negotiating terms, selecting tracks, managing royalties, and assessing audience engagement for fair compensation and impact evaluation.

The main challenge was:

#### *How can Sonic Hub manage both structured business data and flexible audience interaction data while extracting meaningful insights that support business growth?*

To address this challenge, this project implements a Polyglot Persistence Architecture, combining SQL Server and MongoDB to create a unified analytical environment.

The project further investigates an important business question:

#### *Does audience engagement with influencer content directly impact the royalty revenue generated by Sonic Hub's music catalogue?*

### Business Objectives
This project aims to:
- Design a scalable relational database for music operations and royalty management.
- Build a flexible NoSQL database to store audience engagement and feedback.
- Integrate SQL and MongoDB datasets using Python.
- Analyse the relationship between audience engagement and royalty revenue.
- Identify high-performing music tracks based on both commercial and audience performance.
- Demonstrate how Polyglot Persistence can improve business analytics in the music industry.

## 2. Database Architecture
The project utilises two distinct database technologies, tailored to the nature of the data.

### SQL Server (Structured Data)
#### Stores:
Clients
Music Track Catalogue
Reservations
Composers
Copyright Claims
Royalty Management
Collaboration Agreements

#### SQL Server was selected because these business functions require:
- ACID transactions
- Strong consistency
- Referential integrity
- Transaction reliability

For example, royalty payments and collaboration agreements must remain accurate even if system failures occur.

### MongoDB (Semi-Structured Data)
#### Stores:

#### Engagement Metrics:
- Likes
- Shares
- Comments
- Views

#### Viewer Feedback
- Overall Quality
- Creativity
- Relevance
- Engagement
- Recommendations
- Comments

MongoDB was selected because audience interactions vary significantly in structure and volume. Its schema-less architecture allows Sonic Hub to scale audience analytics without requiring changes to rigid database schemas.

## 3. Methodology
### Step 1: Relational Database Design

A relational database was designed using SQL Server to manage:

- Client information
- Music tracks
- Reservations
- Royalties
- Copyright claims
- Collaboration agreements

The database was normalised to Third Normal Form (3NF) to:

- Eliminate redundancy
- Improve maintainability
- Preserve data integrity

### Step 2: NoSQL Implementation

MongoDB collections were created for:

Engagement Metrics: Likes, Shares, Views, Comments
Viewer Feedback: Overall Quality, Creativity, Engagement, Recommendations, Comments

The document model provides flexibility for storing evolving audience behaviour data.

### Step 3: Data Integration
Data was extracted from:

SQL Server:
- Revenue earned
- Royalty payments
- Music track information

MongoDB:
- Engagement metrics
- Audience feedback

The datasets were:
- Cleaned
- Transformed
- Merged using Python and Pandas

### Step 4: Statistical Analysis

The merged dataset was analysed using:
- Scatter plots
- Correlation analysis
- Linear Regression
- Comparative visualisations

The objective was to quantify the relationship between:
#### Audience Engagement → Royalty Revenue

### 4. Polyglot Persistence Improved Analytical Capability

The integration of SQL and MongoDB allowed Sonic Hub to:

- Combine transactional and behavioural data.
- Analyse customer sentiment alongside revenue.
- Support future recommendation systems.

Instead of storing data in isolated systems, Sonic Hub can now make decisions using a unified business view.

## Installation and Setup

To run this project locally, ensure that both **SQL Server** and **MongoDB** are installed and configured on your machine.

The project uses Python for data extraction, integration, analysis, and visualisation. Install the required libraries using:

<pre>
pip install pandas numpy pymongo matplotlib statsmodels pyodbc faker
</pre>

### Workflow
1. Create and populate the relational database in SQL Server.
2. Create the MongoDB collections for engagement metrics and audience feedback.
3. Configure database connection parameters in the Python scripts.
4. Run the Python notebooks/scripts to:
- Extract data from SQL Server and MongoDB
- Clean and merge the datasets
- Perform regression and correlation analysis
- Generate visualisations and business insights

This project demonstrates how Polyglot Persistence can bridge structured business operations with flexible audience analytics, enabling Sonic Hub Studios to transform raw data into actionable business insights that drive revenue growth and strategic decision-making.
